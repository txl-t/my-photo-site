# 产品需求文档 (PRD)

## 1. 项目概述
本项目是一个极简风格的摄影作品集网站，旨在展示创作者的日常摄影作品，为访客提供治愈系内容，同时方便潜在合作方了解创作者的视觉风格，并支持管理员通过简单方式维护网站内容。

## 2. 用户角色

### 2.1 访客 (治愈系内容爱好者)
- **需求**：浏览日常摄影作品，获取生活解压阀，感受"原来我身边也有这样的美"。
- **行为**：访问首页，浏览作品展示区，点击查看大图，可能会点赞喜欢的作品。

### 2.2 访客 (潜在合作方)
- **需求**：快速了解创作者的视觉风格与真实性，评估合作可能性。
- **行为**：访问首页，浏览完整作品展示，查看联系信息。

### 2.3 管理员 (创作者)
- **需求**：通过手机或电脑直接上传照片、修改文案，无需编程即可维护整个网站。
- **行为**：更新作品图片，修改网站文案，管理内容。

## 3. 核心功能模块

| 模块名称 | 功能描述 | 技术难点/亮点 |
|---------|---------|--------------|
| 首页 (index.html) | 极简Hero区展示个人标语+精选3张作品预览+作者卡片 | 纯CSS实现滚动隐藏式导航，localStorage存储阅读偏好。 |
| 作品展示区 | 瀑布流布局，点击放大查看，点赞本地持久化 | 无后端架构下用localStorage实现跨页面点赞数据同步 |
| 联系页 (contact.html) | 极简邮箱链接+邮件订阅弹窗。 | 纯前端实现订阅功能，数据存储在浏览器本地 |
| 全局功能 | 响应式设计（PC/手机/平板）、图片懒加载 | 原生JS实现IntersectionObserver，无第三方库 |

## 4. 功能详细说明

### 4.1 首页 (index.html)
- **Hero区**：展示个人标语和背景图片
- **作品预览**：精选3张作品展示
- **作者卡片**：包含头像和简短介绍
- **导航栏**：纯CSS实现的滚动隐藏式导航
- **阅读偏好存储**：使用localStorage保存用户的阅读偏好

### 4.2 作品展示区
- **瀑布流布局**：使用CSS Grid/Flexbox实现
- **图片放大**：点击作品可放大查看
- **点赞功能**：支持点赞，数据通过localStorage持久化
- **跨页面数据同步**：不同页面间点赞数据保持一致

### 4.3 联系页 (contact.html)
- **邮箱链接**：极简设计的邮箱联系方式
- **邮件订阅**：弹窗形式的邮件订阅功能
- **本地数据存储**：订阅数据存储在浏览器本地

### 4.4 全局功能
- **响应式设计**：适配PC、手机、平板等不同设备
- **图片懒加载**：使用原生JS的IntersectionObserver API实现
- **零第三方依赖**：纯原生技术实现，无外部库

## 5. 技术要求

### 5.1 技术栈
- **基础技术**：纯 HTML5 + CSS3 + 原生 JavaScript (ES6)
- **样式方案**：原生 CSS + CSS Grid/Flexbox
- **数据存储**：浏览器 localStorage
- **部署平台**：GitHub Pages

### 5.2 性能要求
- 页面加载时间 < 2秒
- 图片懒加载，提升初始加载速度
- 响应式设计，适配各种设备尺寸

### 5.3 兼容性要求
- 支持 Chrome、Firefox、Safari、Edge 等现代浏览器
- 支持移动端浏览器

## 6. 设计要求

### 6.1 视觉风格
- 极简主义设计
- 突出摄影作品本身
- 简洁、干净的界面

### 6.2 色彩方案
- 以黑白灰为主色调
- 突出作品色彩
- 简洁、专业的视觉效果

### 6.3 排版
- 清晰的层级结构
- 易读的字体大小
- 合理的行间距和字间距

## 7. 内容要求

### 7.1 图片要求
- 作品图片：建议单张 < 300KB
- 支持JPG、PNG格式
- 最多支持50张作品

### 7.2 文案要求
- 简洁明了
- 突出个人风格
- 易于维护和更新
